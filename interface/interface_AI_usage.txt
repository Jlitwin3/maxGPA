AI USAGE DOCUMENTATION
MaxGPA Project
Last Updated: May 4, 2026

--------------------------------------------------
OVERVIEW
--------------------------------------------------

Artificial Intelligence tools were used throughout the development of this project to assist 
with implementation, debugging, and design refinement. AI was used as a support tool to 
accelerate development and clarify technical concepts, but all major design decisions, 
architecture, and final code integration were performed by the team members.

AI contributions were primarily focused on:
- Debugging and troubleshooting
- UI behavior refinement
- Code structuring and organization
- Documentation assistance for clean formatting of comments and manuals.

--------------------------------------------------
FRONTEND (Student Interface)
--------------------------------------------------

AI was heavily utilized during frontend development to assist with JavaScript logic and UI behavior.

Key areas of assistance:

1. Event-Driven UI Design
AI helped refine the structure of event listeners and state-driven rendering. This included guidance on:
- Managing a centralized state object
- Re-rendering UI components after state updates
- Handling user interactions such as clicks, dropdown changes, and button actions

2. Dynamic Rendering Logic
AI assisted in structuring functions responsible for:
- Generating the schedule grid dynamically
- Updating course selections and instructor selections
- Maintaining synchronization between UI components (grid, selection area, instructor panel)

3. Layout and UI Refinement
AI provided guidance on restructuring the layout to improve usability, including:
- Separating instructor selection from the side panel
- Positioning the instructor panel next to the grid
- Using flexbox to organize the interface cleanly

4. Debugging and Issue Resolution
AI was used to diagnose and fix issues such as:
- UI elements not updating correctly
- Incorrect rendering order
- Event listeners not firing as expected
- State inconsistencies after user actions

5. Incremental Feature Development
AI supported step-by-step feature implementation, including:
- Adding instructor selection workflows
- Enforcing constraints (e.g., max 4 classes per cell)
- Preventing duplicate course selection


While AI was a useful development aid, it did not replace human decision-making. 
The following aspects were entirely determined by the team:

- Overall system architecture
- Feature selection and prioritization
- UI/UX design decisions
- Integration between frontend, backend, and database
- Final debugging and validation

AI-generated suggestions were frequently modified, adapted, or rejected based on project 
requirements and constraints.
